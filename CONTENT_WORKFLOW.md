# 沉浸式深度导读 · 内容提取工作流
# Content Extraction Workflow v1.0

> 本文档定义了从原书PDF到高质量data.js的标准化流程。
> 核心原则：**内容深度 > 交互花样**。没有深度内容的交互只是空壳。

---

## 流程概览

```
Step 1: 检视阅读（30分钟）
  → 产出: BOOK_OVERVIEW.md（全书骨架）
  → 你审核，确认方向

Step 2: 逐章分析性阅读（每章15-20分钟，最耗时）
  → 产出: CHAPTER_ANALYSIS.md（每章完整论证链）
  → 你审核，补充你的理解和联想

Step 3: 批判性综合（20分钟）
  → 产出: SYNTHESIS.md（跨章联结、批判性评价、启发）
  → 你审核，加入你的个人视角

Step 4: 格式化为data.js（30分钟）
  → 产出: data.js + custom.js
  → 构建、检查、验收
```

**关键规则**：每一步都必须你审核通过才进下一步。不要跳步。

## Step 1: 检视阅读

### 给Claude Code的Prompt

```
请阅读项目根目录的 DESIGN_SPEC.md，了解引擎的数据格式要求。

然后执行以下任务：

**任务：对《人类简史》进行检视阅读**

请阅读项目目录中《人类简史》的PDF原书，完成以下工作：

1. **全书结构**：列出全书的部分划分和每章标题，标注每章大约的页数/篇幅
2. **核心论点**：用3-5句话概括全书的中心论点（不是主题描述，是作者要证明什么）
3. **作者试图解决的问题**：这本书回答了什么核心问题？
4. **论证策略**：作者用什么方式论证？（历史叙事？数据？案例对比？思想实验？）
5. **关键术语清单**：列出10-15个作者赋予特定含义的术语（如"认知革命""想象的秩序""奢侈品陷阱"等），给出作者自己的定义
6. **全书情绪节奏**：哪些章节是震撼性的？哪些是论证性的？哪些是反思性的？（这会影响后续的调色板和交互设计）

将结果输出为 BOOK_OVERVIEW.md，保存在 src/books/sapiens/ 目录下。

**重要**：
- 所有内容必须基于PDF原文，不要凭记忆补充
- 引述关键段落时标注页码或章节位置
- 如果某些内容在PDF中不清晰，标注[待确认]
- 不要生成任何代码
```

### 你要审核什么
- 核心论点是否准确（不是"讲述了人类历史"这种废话，而是"人类的成功源于虚构故事的能力，但这种成功并未带来更多幸福"这样锐利的概括）
- 关键术语是否完整，定义是否是作者本人的用法
- 情绪节奏的判断是否合理

## Step 2: 逐章分析性阅读

### 给Claude Code的Prompt（以Part I为例，其他Part同理）

```
请阅读 src/books/sapiens/BOOK_OVERVIEW.md 了解全书骨架。

**任务：对《人类简史》Part I（第1-4章）进行分析性阅读**

请逐章阅读PDF原文，对每一章输出以下内容：

### 每章必须提取的7个要素：

1. **章节核心论点**（1-2句）：这一章要证明什么？
2. **完整论证链**（这是最重要的部分）：
   - 前提/假设 → 证据/案例 → 推导过程 → 结论
   - 如果一章有多条论证线，全部列出
   - 每个论证步骤都要包含原书的具体案例、数据或例子
3. **关键案例/故事**（完整复述，不是概括）：
   - 赫拉利用了哪些具体的历史案例、数据、思想实验来支撑论证？
   - 每个案例的完整叙述（保留故事性，这是原书的魅力所在）
   - 标注案例在原书中的位置
4. **数据和数字**：
   - 原书中引用的所有具体数据（人口数字、时间点、比例等）
   - 这些数据服务于哪个论证
5. **反直觉洞察**：
   - 这一章中最颠覆常识的观点是什么？
   - 赫拉利是怎么让读者从"不可能"到"原来如此"的？
6. **章节间的逻辑衔接**：
   - 这一章是怎么从上一章的结论出发的？
   - 这一章的结论如何为下一章铺垫？
7. **可视化/交互机会**：
   - 这一章的哪些内容适合做成交互组件？（对比表、时间轴、数据图、模拟器等）
   - 每个建议要说明"这个交互帮助读者理解什么"

将结果输出为 CHAPTER_ANALYSIS_PART1.md，保存在 src/books/sapiens/ 目录下。

**重要**：
- 论证链必须完整，不能跳步。如果赫拉利用了3个案例支撑一个论点，3个都要写
- 案例要保留叙事性，不要压缩成一句话摘要。"标致公司的例子"不够，要完整讲清楚赫拉利是怎么用这个例子论证"公司是虚构实体"的
- 每个要素都标注来源章节和大致页码
- 如果某些论证你觉得赫拉利没有充分支撑（证据不足或逻辑跳跃），也标注出来——这是批判性阅读的一部分
- 不要生成任何代码
```

### 你要审核什么
- 论证链是否完整，是否有跳步
- 案例是否有叙事感（读起来像在讲故事，不是在列提纲）
- 数据是否准确（核对几个关键数字）
- "可视化/交互机会"的建议是否合理

### 注意
- Part I做完审核通过后，再用同样的Prompt处理Part II、III、IV
- 每个Part是一次独立对话，避免上下文过长

## Step 3: 批判性综合

### 给Claude Code的Prompt

```
请阅读 src/books/sapiens/ 目录下的所有分析文档：
- BOOK_OVERVIEW.md
- CHAPTER_ANALYSIS_PART1.md
- CHAPTER_ANALYSIS_PART2.md
- CHAPTER_ANALYSIS_PART3.md
- CHAPTER_ANALYSIS_PART4.md

**任务：批判性综合**

基于以上分析，输出以下内容：

1. **全书论证主线重建**：
   - 用一条逻辑链串起全书：认知革命→农业革命→融合统一→科学革命→未来
   - 每个环节之间的因果关系是什么？
   - 赫拉利的"暗线"是什么？（每次"进步"的代价）

2. **跨章概念联结**：
   - 哪些概念在不同部分反复出现但含义演变了？
   - 哪些早期的论证在后期被推翻或深化了？
   - 画一张概念关系图（文字描述即可）

3. **批判性评价**：
   - 赫拉利最有说服力的论证是哪3个？为什么？
   - 赫拉利最薄弱的论证是哪3个？证据不足还是逻辑跳跃？
   - 学术界对这本书的主要批评是什么？（如果你知道的话）

4. **"超越原书"的增值内容**：
   - 赫拉利的论点与当下AI革命的联系
   - 与中国历史的对照（秦统一 vs 帝国融合、科举 vs 想象的秩序等）
   - 与Will的CFO/投资视角的联系（公司作为虚构实体、货币的信用本质、资本主义的信贷引擎等）

5. **读完应该能回答的10个问题**：
   - 设计10个"分析性阅读完成度"检验问题
   - 这些问题不是记忆性的（"认知革命发生在多少年前"），而是理解性的（"为什么赫拉利认为农业革命是'史上最大骗局'？你同意吗？"）

将结果输出为 SYNTHESIS.md，保存在 src/books/sapiens/ 目录下。
不要生成任何代码。
```

### 你要审核什么
- 跨章联结是否真有洞察（不是简单罗列）
- 批判性评价是否言之有据（不是为了批判而批判）
- "增值内容"中与你个人视角的联系是否准确（你可以补充修改）
- 10个检验问题是否真的检验了深度理解

## Step 4: 格式化为data.js

### 给Claude Code的Prompt

```
请阅读以下文件：
1. DESIGN_SPEC.md（数据格式规范）
2. src/books/sapiens/BOOK_OVERVIEW.md
3. src/books/sapiens/CHAPTER_ANALYSIS_PART1-4.md
4. src/books/sapiens/SYNTHESIS.md
5. 当前的 src/books/sapiens/data.js（了解现有结构）
6. 当前的 src/books/sapiens/custom.js（了解已有的自定义组件）

**任务：基于分析性阅读成果，重写 data.js**

要求：
1. **每个scene的narrative不少于6段**，每段300-500字，保留论证的完整逻辑链和关键案例的叙事性
2. **每个scene至少3个deepDives**，每个deepDive的chain（前提→证据→结论）都要有实质内容，不是一句话概括
3. **heroQuote选用原书中最震撼的原句**（标注来源章节）
4. **extendedReading填充完整**，不是空数组
5. **keyTakeaway必须是理解性总结**，不是记忆性摘要
6. **至少规划3个新的自定义交互组件**（写在custom.js中），每个要回答"它帮助读者理解什么"：
   - 参考CHAPTER_ANALYSIS中的"可视化/交互机会"建议
   - 参考已有的Dunbar圈层、时间线探索器等成功案例
7. **SYNTHESIS.md中的批判性内容**要融入：跨章联结做成知识地图、批判性评价做成aiReflection、10个检验问题做成Active Recall
8. **数据格式严格遵守DESIGN_SPEC.md第三节**的模板，特别是heroQuote用字符串、deepDives用chain嵌套、extendedReading用字符串数组

先输出 data.js，然后输出 custom.js。
然后运行 node build.js 构建。
构建成功后，用DESIGN_SPEC第五节的10项清单逐一验证。
报告验证结果。
```

## 持续使用指南

当你要给下一本书（比如《思考快与慢》）做同样的深度处理时：

1. 确保原书PDF在项目目录中
2. 按Step 1→2→3→4依次执行
3. 每一步都是独立的Code对话
4. 每次对话的第一句话：
   > 请先阅读 DESIGN_SPEC.md 和 PROJECT_STATUS.md，然后阅读 src/books/thinking-fast-slow/ 目录下已有的分析文档（如果有的话）。

这套流程稳定后，可以封装成Claude Skill，进一步自动化。

## 时间估算（单本书）

| 步骤 | AI工作时间 | 你的审核时间 | 总计 |
|------|-----------|------------|------|
| Step 1 检视阅读 | 10-15分钟 | 10分钟 | ~25分钟 |
| Step 2 分析性阅读（4部分） | 4×20分钟 | 4×15分钟 | ~2.5小时 |
| Step 3 批判性综合 | 15分钟 | 20分钟 | ~35分钟 |
| Step 4 格式化+构建 | 30分钟 | 15分钟 | ~45分钟 |
| **总计** | | | **~4-5小时** |

这比你之前花两天反复调交互要高效得多，而且产出质量会有本质差别。
