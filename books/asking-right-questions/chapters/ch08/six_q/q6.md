# Q6 · 我如何实践

针对 reader_persona —— **知识工作者,追求系统化的学习与工作方法**。本章把"证据效力评估"具体到五类高频出现但可靠度有限的证据类型。以下五条把这些评估工具嵌入日常信息消费与决策工作流。

## 1. "证据类型红黄绿"速评法:读任何推理性内容时跑一次

**建议是什么**:阅读任何推理性文档(内部报告、行业研究、新闻分析、战略 deck、咨询建议)时,把文中的每条支撑证据按三色分类——

- **红色**(高度警惕):直觉断言("我感觉…""凭经验判断…"没有可核查的基础)、营销式当事人证词、单一典型案例、网络"专家"引用链、不署名来源、引用套引用。
- **黄色**(可参考但要追问):个人经历、具名典型案例(含可追溯信息)、明确资质的专家意见。对每条黄色证据跑一次"专家四问":专长相关吗?有第一手资料吗?有偏见吗?可靠性记录如何?
- **绿色**(可信度较高):经同行评审的研究、来自第一手数据的统计、有公开方法论的调查、多个独立来源的交叉印证。

如果一份文档的关键结论**仅靠红色或黄色证据**支撑,结论本身的可信度不超过证据的颜色等级。

**触发场景**:
- 跨部门提案和战略建议书的评审。
- 咨询公司给的推荐报告(特别是含大量 case study 或客户证词的 deck)。
- 财经 / 科技 / 医疗新闻里"专家说"的段落。
- 社交媒体上的"干货"文章和 KOL 推荐。

**执行时注意事项**:
- 不要一看到"红色"就全盘否定——本章也强调"要给所有论证它们应得的机会"。红色证据可以作为**进一步调查的线索**,不可作为结论依据。
- **总是反问一句"绿色证据在哪里?"**——如果文档从不引用研究或第一手数据,只靠红黄证据堆砌,就要对其结论做折扣处理。
- 在自己的汇报文档里,主动标注证据等级(至少区分"此处为个人观察"vs"此处为第三方研究")——这是本章"作者应主动披露证据"伦理的具体落地。

## 2. "专家引用四维追踪表":采信任何"专家说"前必填

**建议是什么**:每当文档中出现"XX 专家说 / XX 机构研究显示 / 某报报道",在接受其作为证据前,填一张**四维追踪表**——

| 专家 / 来源 | 1. 专长是否与本论题吻合? | 2. 这是第一手还是第二手? | 3. 可能的偏见(利益、立场、机构关系)? | 4. 历史可靠性记录? | 采信度(高/中/低) |

四列都**必须填**。第 2 列如果是第二手,要能沿引证链追到原始第一手;如果追不到(匿名来源、引用套引用、原始链路断裂),该条证据**自动降为低采信**。

**触发场景**:
- 投资决策前对行业研究的采信评估。
- 医疗健康建议前的来源评估(特别是跨境电商产品说明、功能性食品、保健品)。
- 政策解读、合规分析、法规变更公告的采信。
- 跨境新闻、翻译报道、二次加工信息的源头评估。

**执行时注意事项**:
- "专长相关性"比"专家名气"重要——诺贝尔奖得主在自己领域外的观点不自动有效。托马斯·沃森和扎努克两例就是本章的明证。
- **跨利益互证**:本章给出的一条核心互证方法——"检查持不同观点、不同价值观和利益的各路专家是否同意这个断言"——在填表时应单独加一栏"反向立场专家意见"。如果各方一致,采信度高;如果同立场一致但跨立场分歧,采信度低。
- AI 生成内容时代,四维表要补一列"是否可能是 AI 生成 / 合成"——特别是来自社交媒体、非核心媒体、聚合网站的"专家表态"。

## 3. "典型案例解毒":重大决策前强制做一次反例搜索

**建议是什么**:当一个决策(业务投资、人事任命、新业务线开拓、合作伙伴选择)主要由**几个典型成功案例**驱动时,在表态前强制做一次**反例搜索**——

- 列出让你倾向该决策的 1-3 个成功案例。
- 对每个成功案例,主动搜索**同类但失败的案例**:"在类似条件下尝试过、失败了的有哪些?"
- 对比两组案例,问:"我的决策是基于成功案例的代表性,还是基于它们**特别吸引我的生动性**?"
- 如果在反例搜索中找不到失败案例(这本身就可疑),问:"为什么失败案例没有被保留下来?是因为真的没发生,还是因为被幸存者偏差筛掉了?"

**触发场景**:
- 跨境电商新市场进入决策(大家都会举几个成功品牌,很少举大量失败撤退的)。
- 新技术或新业务模式的投资评估(宣传总是几个 unicorn,鲜有人展示 99% 的失败)。
- 组织变革参考(人们爱引用一两家转型成功公司,忽略大量转型失败的)。
- 人事选拔中的"他很像我们当年某位大牛"类比。

**执行时注意事项**:
- 反例搜索要主动、要刻意——**幸存者偏差**(survivorship bias)是典型案例不可靠的根源。Abraham Wald 飞机弹孔分析的故事是这条提醒的经典原型:活着回来的飞机弹孔分布不是"该加固的位置",而是"就算被击中也没坠毁的位置"——真正需要加固的是没飞回来的飞机的被击中位置。
- 本章给的"119 名无辜死囚被释放"示例是一个**反例打破典型案例**的典范——可以把它当做自己这一思维动作的参考模板。
- 反例不必证明你的决策错误,但它让你知道决策的**失败概率与条件**——决策仍可做,但需基于失败概率做风险准备。

## 4. "当事人证词四问剥离":面对所有推荐 / 背书前

**建议是什么**:遇到任何当事人证词(产品推荐、服务背书、书评、咨询客户 case study、候选人推荐信、供应商客户案例),在采信前跑本章的**四问剥离**——

1. **选择性**:"这份证词是如何被挑选出来的?没被挑选出来的同类证词说什么?"
2. **个人兴趣**:"证词提供者是否从被推荐对象中获得金钱、声誉、人际或职业利益?"
3. **省略信息**:"这份证词里缺失哪些我做判断所需的信息?(数量、条件、时长、对比、失败率)"
4. **人为因素**:"这份证词的可信度有多少来自证词者的人设 / 情绪 / 视觉呈现,而非证词本身?"

每一问都能把证词的"表面可信度"折扣一次。四问跑完,证词通常只剩下"值得进一步调查"的价值。

**触发场景**:
- 供应商选择时审查他们的客户案例库。
- 雇佣面试时审查候选人的推荐信和过去业绩描述。
- 购买商业服务(咨询、审计、培训)前审查服务商的案例展示。
- 书籍、课程、工具选择前审查各类用户证词。

**执行时注意事项**:
- 本章"医药公司付津贴让医生开某品牌药"是"个人兴趣"维度的原型案例——在 B2B 合作、咨询推荐、行业联盟中,这种结构性利益关系无处不在,必须主动搜索。
- 面对证词,问"如果我能选出最佳证词,这份是不是已经是最佳?如果是,代表性就极可疑"——选择性问题的反向思考。
- 不要过度否定——合法的客户案例(含具体数据、可被追溯的项目、可被反向访问的客户方)仍然是有参考价值的;只有**难以被核实的包装性证词**才需要全盘折扣。

## 5. "学术级证据写作"习惯:所有重要对外文档

**建议是什么**:对所有**将长期影响他人决策**的文档(战略备忘录、投资建议、研究报告、白皮书),在定稿前做一次**证据升级审查**——

- 找出文档中所有支撑关键论证的证据,分类到本章五种类型 + 研究证据。
- 对每个"五类弱证据"支撑,问:"有没有办法把它替换或补充为研究证据 / 第一手数据?"
- 如果无法替换,**主动披露该证据的局限**——不是掩盖而是让决策者知情。
- 最终文档应在相关章节或附录里附上:
  - 主要数据来源清单(出处、时间、方法论)。
  - 主要研究引用清单。
  - 使用当事人证词 / 典型案例的部分,明确标注其**非代表性**。
  - 依赖专家意见的部分,披露专家背景与潜在偏见。

**触发场景**:
- 给决策层 / 董事会的战略汇报。
- 投资委员会的立项文档。
- 对外发布的研究报告、行业白皮书、市场分析。
- 需要长期保存的决策备忘录(比如年度战略、重大合作决定)。

**执行时注意事项**:
- 本章明言:"可公开验证、依据科学方法进行并且在出版之前接受同行评价的研究被赋予较高的价值和意义"——这一标准是写作证据层级的**北极星**。实战中很难 100% 达到,但用作校准方向永远正确。
- 披露证据局限不会让你的文档显得弱,反而让它显得**诚实、严谨、值得信赖**——这是本章"期待批判性读者"一节的精神。决策者常常比作者更懂得证据的限度,隐藏局限只会让他们怀疑全篇。
- 把证据审查做成模板——每次写类似文档时从模板起步,让习惯跨文档沉淀,不因时间压力省略。

---

**元原则**:ch08 最深的价值不是让你"不相信直觉 / 经历 / 典型案例 / 证词 / 专家",而是让你对每种证据**匹配正确的采信水平**——直觉可以作为线索,经历可以作为可能性,典型案例可以作为兴趣入口,当事人证词可以作为调查起点,专家意见可以作为参考——但没有任何一类可以单独承载高风险决策。知识工作者的成熟标志之一,就是**对同一件事的不同来源有不同的采信预期**,并且能在"追求完美证据"与"无穷搁置决策"之间找到**基于争议性与风险的合理带宽**——这正是本章"结论或理由争议性越大,花在提供证据上的时间就应该越多"背后的真正智慧。
