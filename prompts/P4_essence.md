# P4 · 全书精华本 Prompt

**使用场景**:所有章节 insights.md → book_essence.md
**建议模型**:Claude Opus 4.7
**effort**:xhigh

---

## 设计目标

**这是"替代阅读原书"目标的主要载体。**

读完 book_essence.md,读者应该:
- 掌握了全书所有核心论点及其论证过程
- 理解了作者思想的适用边界
- 能够在实际场景中调用本书的思维工具
- 不再需要读原书(除非要做学术引用或品味文字)

---

## 关键设计:分章组织 vs 按主题重组

**选择:分章组织。**

原因:
1. 保留作者的原始论证节奏
2. 便于读者回查原书对应章节
3. 按主题重组容易丢失章节间的递进逻辑

---

## Prompt 模板(分章处理,每章独立一轮)

```
任务:为本书第 {N} 章撰写精华本章节内容。

要求:
1. 字数 800-1200 字
2. 读者看完这一节,相当于读完了原书对应章节
3. 必须包含:
   - 本章核心主张(完整复述)
   - 作者的主要论证(事实+逻辑链)
   - 本章的实践价值(怎么用)
4. 可以适度补充 Q3 的反例局限,但不要让批判压过原作
5. 语言风格:保留作者的论证风格,但用更紧凑的表达
6. 关键金句处,用引号直接保留作者原文(每章 1-2 句即可)

输入:
- ch{N}/insights.md(尤其是 Q1、Q2、Q6)
- ch{N}/quotes.md(原文金句)
- ch{N}/extract.json 的 key_data 和 key_terms

输出结构:

## 第 {N} 章 · {章节标题}

### 核心主张
{该章作者的核心观点,用自己的话复述,保留关键术语}

### 论证过程
{作者如何支撑核心主张,包含关键论据和数据}

### 实践启示
{本章对读者的可操作价值}

### 作者原声
> "{原文金句 1}"
> "{原文金句 2,如有}"
```

## 组装完整的 book_essence.md

所有章节单独生成后,按以下结构组装:

```markdown
# {书名} · 精华本

> 替代阅读原书的认知转化产出
> 原书 {N} 章,精华本约 {字数} 字

## 如何使用本精华本

1. 第一次阅读:从头到尾顺序阅读,建立全书认知框架
2. 复习使用:配合 book_skeleton.md 快速回顾核心
3. 检索使用:配合 themes/ 索引按主题查找
4. 深度追问:每章末尾标注了对应 insights.md 路径,需要深挖时打开
```

## 调用示例

```
请生成 intelligent-investor 的精华本:

阶段 1:逐章生成(并行,但每章独立 prompt)
对 ch01 到 ch20,分别读取 insights.md,按 prompts/P4_essence.md 的要求,
生成每章的精华内容,暂存为 chapters/chXX/essence_section.md

阶段 2:组装
把所有 essence_section.md 按章节顺序合并,加上开篇说明、目录、全书总结,
保存为 books/intelligent-investor/book_essence.md

阶段 3:校验
字数不低于 15000,不高于 25000
每章精华 800-1200 字
每章至少含 1 条作者原文引用
```
