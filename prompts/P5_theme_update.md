# P5 · 主题索引更新 Prompt

**使用场景**:新书处理完成 → 更新 themes/ 下相关主题文件
**建议模型**:Claude Sonnet 4.6
**effort**:medium

---

## 设计目标

让图书馆具备**跨书主题检索**能力。

例子:你查询"安全边际"这个主题,系统应该能告诉你:
- 格雷厄姆《聪明的投资者》第 20 章主讲
- 霍华德·马克斯《投资最重要的事》也有讨论(不同视角)
- 塞思·卡拉曼《安全边际》专著

---

## 主题分类体系(预设,可扩展)

**投资类**:价值投资 / 成长投资 / 宏观分析 / 行为金融 / 风险管理 / 估值方法
**认知类**:认知偏差 / 心智模型 / 决策理论 / 批判性思维 / 元认知
**历史类**:中国古代史 / 现代史 / 经济史 / 科技史 / 思想史
**地缘类**:大国博弈 / 国际关系 / 全球化
**管理类**:财务管理 / 战略管理 / 组织行为 / 领导力
**哲学类**:科学哲学 / 伦理学 / 政治哲学
**(其他按需扩展)**

## 单个主题文件的结构

每个主题文件 `themes/{主题名}.md` 采用统一结构:

```markdown
# 主题:{主题名}

> 跨书主题索引 | 最后更新:{timestamp}

## 主题定义
{一段 200-300 字的主题界定}

## 本主题下的核心问题
- {问题 1}
- {问题 2}
- ...

## 相关书籍与关键章节

### {书名 1}(作者,年份)
**核心贡献**:{该书对本主题贡献了什么,2-3 句话}

**关键章节**:
- [第 X 章:{章节名}](../books/{book-slug}/chapters/chXX/insights.md)
- [第 Y 章:{章节名}](...)

**代表观点**:
> "{原文金句}"

---

### {书名 2}(作者,年份)
(类推)

---

## 不同作者的观点比较
| 议题 | {作者 A} | {作者 B} | {作者 C} |
|------|---------|---------|---------|
| ... | ... | ... | ... |

## 个人综合判断
{在阅读了多位作者后,读者自己对这个主题的综合认识,可留空待用户填写}
```

## Prompt 模板

```
任务:基于新完成的一本书,更新相关主题索引。

步骤:
1. 读取 books/{book-slug}/book_skeleton.md 和 meta.json
2. 识别本书主要涉及的 2-4 个主题(从预设分类体系中选,必要时新建)
3. 对每个主题:
   a. 若 themes/{主题名}.md 已存在:读取,在"相关书籍与关键章节"节追加本书条目
   b. 若不存在:创建新文件,按模板填充
4. 为本书生成每个主题的贡献描述(2-3 句话)
5. 识别本书中与已有书籍观点有对比价值的议题,在"不同作者的观点比较"表中新增行
6. 每次更新后,更新文件顶部的 timestamp

输入:
- meta.json
- book_skeleton.md
- book_essence.md
- themes/ 目录下已有的主题文件

严格要求:
- 不得覆盖已有主题文件的内容,只做追加和更新
- 新增条目必须带章节 insights.md 的相对路径链接
- 原文金句必须来自 book_quotes.md,不得编造

输出:
- 列出本次更新涉及的所有 themes/ 文件
- 每个文件说明新增了什么
```

## 调用示例

```
intelligent-investor 处理完毕,请更新主题索引:

1. 读取 books/intelligent-investor/book_skeleton.md 和 meta.json
2. 识别涉及的主题(预期:价值投资、风险管理、行为金融、估值方法)
3. 对每个主题,按 prompts/P5_theme_update.md 的要求更新 themes/ 下对应文件
4. 若有新主题,从预设分类体系中选择合适的名称,不要乱造新类
5. 完成后输出一个更新报告:本次新增了哪些条目、修改了哪些文件
```
